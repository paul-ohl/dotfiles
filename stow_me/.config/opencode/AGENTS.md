# Global OpenCode Rules

## Skill Discovery

When a situation arises where a specialized skill could meaningfully help — such as
working with a specific framework, debugging a complex issue, writing tests, reviewing
code, or any domain-specific task — proactively load the `find-skills` skill and use
it to search for a relevant installable skill.

Situations that should trigger a skill search:
- The user asks for help with a specific technology, framework, or tool
- A task involves a domain you don't have deep specialized knowledge about
- The user expresses a recurring workflow need (e.g. "I always have to...")
- You are about to do something complex where a community skill likely exists

When you find a relevant skill, present it to the user with the install command
before proceeding with your own approach. Do not silently skip this step.
